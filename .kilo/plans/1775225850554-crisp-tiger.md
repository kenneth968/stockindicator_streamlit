# SMT Trading Signal System - Technical Plan

## Project Overview
Build a futures trading signal system using TradingView data (MNQ/MES) with Streamlit dashboard for live signals and backtesting.

## Data Layer

### tvdatafeed Integration
- **Package**: `tvdatafeed` (PyPI) or fork `rongardF/tvdatafeed` (GitHub)
- **Symbols**: `MNQ1!` (micro NASDAQ), `MES1!` (micro S&P 500)
- **Exchange**: CME (exchange must be specified)
- **Intervals**: 1m (live), 5m/15m/1H/4H/1D (analysis)

### Data Architecture
```
tvdatafeed API
    ↓
OHLCV Cache (SQLite/PostgreSQL for persistence)
    ↓
Signal Engine
```

### Risk Mitigation
- Cache data locally to reduce API calls
- Implement retry with exponential backoff
- Document migration path: Databento, Polygon.io, or broker API (Interactive Brokers, TrendSpider)

## Signal Engine Components

### 1. FVG / IFVG Detector
- **Fair Value Gap (FVG)**: 3-bar pattern where middle bar's range doesn't overlap with adjacent bars
  - Bullish FVG: low[mid] > high[prev]
  - Bearish FVG: high[mid] < low[prev]
- **Inverted FVG (IFVG)**: Opposite direction of trend
- **Fill tracking**: Did price revisit (fill) the FVG zone?

### 2. HTF Bias (Higher Timeframe)
- **4H Analysis**: Last 10 bars - broken above/below last swing high/low
- **1D Analysis**: Daily high/low as directional filter

### 3. Liquidity Sweep Detector
- Track recent highs/lows
- Detect when price sweeps (wicks beyond) liquidity zones

### 4. Order Block (OB) Detector
- Locate lastbearish/bullish order block before FVG
- Track OB-FVG overlap conditions

### 5. SMT Divergence
- Compare MNQ vs MES momentum
- Detect divergence: MNQ makes HH/LH while MES makes LH/HL (or vice versa)

### 6. Session Filter
- **Asia**: 00:00-08:00 UTC
- **London**: 08:00-16:00 UTC  
- **NY**: 13:00-21:00 UTC (overlaps with London)
- Filter signals by active session

### 7. A+ Score (0-100)
Composite confidence score:
- FVG present: +20
- HTF aligned: +20
- Liquidity sweep: +15
- OB-FVG overlap: +15
- SMT divergence: +15
- Session quality: +15

## Technical Decisions

1. **Backtest Range**: 1 year historical data
2. **Data Persistence**: SQLite (local) - simplest setup
3. **Live Polling**: 1 minute interval
4. **Error Handling**: Auto-retry with 3 attempts + exponential backoff
